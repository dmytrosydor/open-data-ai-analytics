## 1. Мета проєкту
Метою роботи є розробка модульної системи для обробки та аналізу відкритих даних про дорожньо-транспортні пригоди. Проєкт демонструє навички роботи з Git-flow, підготовки даних (Data Cleaning), проведення розвідувального аналізу (EDA) та застосування алгоритмів машинного навчання для виявлення закономірностей у міському середовищі.

## 2. Опис реалізованих модулів та структури
Проєкт розділений на функціональні модулі для забезпечення чистоти коду та можливості перевикористання:

- src/data_load.py: Містить функцію getDataFrame(), яка автоматизує зчитування сирих даних.

- src/data_quality.py: Реалізує логіку очищення даних, зокрема видалення записів із пропущеними координатами, що є критичним для геопросторового аналізу.

- notebooks/1.0-eda.ipynb: Інтерактивне середовище, де проведено первинний аналіз (ТОП-5 причин ДТП, аналіз найбільш аварійних вулиць) та побудовано ML-модель.

- src/visualization.py: Скрипт для генерації фінального звіту у вигляді інтерактивної HTML-карти з використанням бібліотеки folium.

## 3. Модель машинного навчання (K-Means)
Для виявлення зон підвищеної небезпеки було застосовано алгоритм кластеризації K-Means.

-Логіка: Алгоритм проаналізував координати 866 інцидентів та автоматично згрупував їх у 5 кластерів.

- Практичне значення: Визначені моделлю центроїди (центри кластерів) є оптимальними точками для дислокації патрульних екіпажів або постів швидкої допомоги. Це дозволяє мінімізувати середній час прибуття на місце події у відповідному секторі міста.

## 4. Робота з Git та вирішення конфліктів
Проєкт реалізовано з використанням гілок (feature branches) та Pull Requests.

- Merge Conflict: Під час інтеграції модулів якості та аналізу було штучно створено конфлікт у файлі README.md.

- Вирішення: Конфлікт було розв'язано вручну шляхом об'єднання інформації про очищення даних та результати моделювання, що зафіксовано у відповідному merge-коміті.

- Реліз: Фінальна версія проєкту позначена тегом v0.1.0.
